Introduction
------------
`vasp_auto` is a Python toolkit that automates preparing, running and parsing
VASP DFT calculations (with a Quantum ESPRESSO engine alongside). It turns a
bare POSCAR into a full input set (INCAR / KPOINTS / POTCAR), launches the run
locally or on a remote HPC machine, parses the output, and writes an Excel
summary. The same engine is driven three ways:

    vasp-auto                CLI            -> vasp_auto.cli:main
    vasp-auto-ui             web UI         -> vasp_auto_ui.server:main
    vasp-auto-setup-remote   remote setup   -> vasp_auto.remote_setup:main

The engine (`src/vasp_auto/`) is the single source of truth; both the CLI and
the web UI call the same functions, so they behave identically.

For the authoritative, always-current module-by-module description and the
list of known problems / next jobs, see CLAUDE.md in this directory. This file
is the directory map; CLAUDE.md is the narrative.

----------------------------------------------------------------------------
The engine: src/vasp_auto/  (23 modules)
----------------------------------------------------------------------------
Entry / orchestration
  cli.py            argparse entry point, orchestrates everything
  calc_types.py     CalcType enum + descriptions + per-type chain inputs
  config_loader.py  config.yaml search, normalisation, per-case merging
  target_utils.py   case-type detection, project vs single mode

Input preparation
  job_manager.py    input-file prep, INCAR templates, dry-run preview
  incar.py          INCAR editing helpers (set_incar_value, spin/MAGMOM)
  kpoints.py        KPOINTS generation (gamma/MP mesh, density, line-mode)
  potcar_finder.py  POTCAR library lookup, variant map, concatenation
  structure.py      pure-Python POSCAR tools (supercell, vacancy, substitution,
                    interstitial, prototypes, supercell matching, adsorbates)
  ase_tools.py      ASE import + bulk/slab/molecule/crystal/nanotube builders
  ml_tools.py       MLIP pre-relax/screening (fairchem UMA/OMat24, EMT demo)

Execution
  runner.py         mpirun invocation + SLURM/PBS submit scripts; remote checks
  remote_setup.py   `vasp-auto-setup-remote`: SSH keys + register a remote
  workflow.py       OUTCAR parsing, error detection + auto-retry, run-one-case
  convergence.py    automatic ENCUT + SIGMA + NELM + KPOINTS scan
  chain.py          chained workflows (converge -> relax -> scf -> dos)

Output / analysis
  parser.py         vasprun.xml parsing (PDOS, bands, k-point labels)
  analysis.py       adsorption energy, vib thermochemistry, d-band, work func
  chgcar.py         CHGCAR/LOCPOT/AECCAR volumetrics + Bader helper
  trajectory.py     XDATCAR/NEB frames -> cartesian animation data
  report.py         per-job Markdown calculation report (--report)
  excel_writer.py   pandas -> styled Excel summary + energy bar chart

Other engine
  qe_tools.py       Quantum ESPRESSO (pw.x) engine: POSCAR -> pw.in, UPF pseudos

----------------------------------------------------------------------------
Notes
----------------------------------------------------------------------------
- src/vasp_auto/ is the active package; the CLI and UI both import it.
- inputs/ is authored by the user; jobs/ is generated output (never committed).
- POTCAR/ is proprietary: never commit it, never print its contents.
- .venv/, venv/, __pycache__/, *.egg-info/ and logs are local/generated.
- Keep this map and CLAUDE.md in sync whenever modules are added or moved.
