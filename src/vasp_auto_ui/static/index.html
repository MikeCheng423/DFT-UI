vasp_auto

Welcome 👋

vasp_auto prepares all VASP input files for you, runs the job, and collects the results. No hand-written INCAR needed.

  1. Pick a working machine, then Build or import a structure and 💾 Save it — that becomes the working case.
  2. Pick a Calculation type — hover anything for an explanation — and press Run.
  3. Chain steps in Workflow (e.g. optimise → SCF → DOS) for a full study in one click.
  4. Watch Results fill in: energies, band gaps, convergence, and a downloadable Excel summary.

✨ AI builder (any AI API)

Describe the structure in plain words and the AI turns it into a build command — bulk, surface, molecule, nanotube, an adsorbate on a slab, or a combined interface. The result loads into the editor; review and 💾 Save when happy. The AI only picks the recipe — coordinates are always built exactly by the code.

Try: bulk fcc copper · CO2 molecule · (5,5) carbon nanotube · graphene under a 2x2 Au fcc(111) slab

🧊 Bulk crystal

A unit cell from tabulated lattice data — just give the element. Loads into the editor.


💠 Prototype crystal (no ASE needed)

Compounds and 2D sheets the element builder can't make: graphene, graphite, rutile/anatase TiO₂, hBN. Tabulated experimental lattice constants — override if needed.


Need a compound that's not in this short list? Use 🗄 Materials Project to search ~150k DFT-relaxed structures and load any of them as a prototype.

🗄 Materials Project (database search)

Search the Materials Project database for a DFT-relaxed prototype. Type a formula (SnO2), a chemical system (Fe-O), or a material ID (mp-2657), then pick a result to load into the editor. Results are ranked by stability (lowest energy above hull first). Needs an MP API key — set the MP_API_KEY environment variable or paste one below.

Applied when you load a result below. Leave blank to load the structure as-is.

🔷 Space-group crystal

Any crystal from its space-group number + a Wyckoff basis (ASE). Give one element and one fractional site per basis line — symmetry generates the rest.


🪨 Surface slab

Cuts a surface from a bulk crystal and adds vacuum above it. The source can also be a saved case POSCAR (e.g. a TiO₂ prototype) — use 📁.


💧 Molecule in a box

🧵 Nanotube

Single-wall (n, m) nanotube, periodic along c with vacuum around it. (5,5) = armchair, (n,0) = zigzag.


📥 Import a structure file

CIF, XYZ, POSCAR and most other formats. The file is read from the working machine chosen at the top — browse pulls it over SSH for a remote.

🧬 Combine two structures

Merge structures with different unit cells — e.g. deposit an Au crystal on a graphite sheet. Host = the structure in the editor; the guest lands gap Å above it, with vacuum Å left on top. Result opens in the editor (save when happy).


🛣 Transition state (NEB) case

Give initial and final structures (case folders or POSCAR files); images are interpolated when run with the neb type.


🧲 Adsorption quick build

From a slab to an adsorption structure: adsorbate above a surface atom, bottom layers frozen. Loads into the editor — review, save, then run relax.


🤖 ML pre-relax (Meta OMat24 / UMA)

Relax the working case with an MLIP before VASP, writing a new _ml case. emt works out of the box (simple metals). The uma-* models need pip install fairchem-core and a Hugging Face login with access to facebook/UMA — otherwise the download fails with a gated-repo error.

DFT engine

VASP needs a paid licence. Quantum ESPRESSO (pw.x) is free and open source. ASE calculator connects to any code ASE can drive (EMT, Quantum ESPRESSO, GPAW, ABINIT, CASTEP, SIESTA, NWChem, MACE …) — pick the calculator and point at its binary.

QE supports scf, relax, vc-relax, dos, bands. NEB/phonon/convergence scans are VASP-only for now.

ASE supports scf and relax only (single-point energy / BFGS). Writes CONTCAR + ase_results.json. Make sure the chosen code (and ASE) is installed where the job runs.

Workflow builder

Chain calculations: each step starts from the previous step's relaxed structure (and charge density where needed). Start from a preset:

Optimise → SCF Optimise → SCF → DOS Optimise → SCF → Bands SCF → DOS Converge → Optimise → SCF → DOS
    Convergence step settings — used by a converge step in the chain

    When the workflow starts with a converge step, a short series of SCF jobs finds the cheapest safe ENCUT / SIGMA / NELM / k-mesh, and those settings are carried into every later step. Leave a field blank to skip that scan (blank ENCUT + SIGMA → just the NELM + k-mesh scan).

    N → NxNxN · hxkxl → exact (use Nx Nx1 for slabs)

    workflow.yaml (advanced)

    ⬇ Excel summary
    No results yet — run a calculation first.
    cross-job tools:

    Minimum energy path —

    The reaction-coordinate (NEB) plot: energy of each image relative to the initial state. The peak is the transition state; Ea is the forward barrier and ΔE the reaction energy.

    Density of states —

    Energies relative to the Fermi level (dashed line). Spin-down is mirrored below zero.

    Projected DOS —

    Per-element s/p/d contributions, relative to the Fermi level (needs LORBIT — the dos template sets it). Restrict to part of the structure with an atom selection.

    Band structure —

    Eigenvalues along the k-path, relative to the Fermi level (dashed). Vertical lines mark the high-symmetry points from the line-mode KPOINTS.

    Charge density —

    Left: planar average along the chosen axis (for LOCPOT this is the work-function profile). Right: a colour map of one plane through the cell — drag the slider to move it. Red/blue maps mean gain/loss for difference data.

    0.5

    Charge-density difference Δρ

    Δρ = ρ(total) − ρ(part 1) − ρ(part 2): where electrons go when fragments bond (e.g. slab+molecule − slab − molecule). All three jobs must use the same cell and grid — run the fragments in the combined cell with frozen positions, calc type charge. Writes CHGCAR_diff next to the total.


    🧪 Property analysis — pick a job

    Derive physical properties from a finished job. Opened by the 🧪 button on a result row. Each button computes one property from that job's output files and needs a matching calculation type — pick the analysis that fits how the job was run: vibrational thermochemistry (ZPE/Gibbs, from a freq job), d-band centre (dos with LORBIT), work function (workfunction), optical absorption (optics), or Bader atomic charges (charge). Same maths as the CLI's --thermo/--d-band/--work-function/--optics-parse/--bader.



    Pick a job folder, then choose an analysis.

    🧗 Adsorption energy

    Eads = E(slab+adsorbate) − E(slab) − scale·E(molecule), from three finished jobs. Use scale 0.5 with an H₂ reference for atomic H. Negative = binds.

    Remote machines
    
        
    Run calculations on another computer or cluster over SSH. When you
          pick a machine in the Calculate tab, vasp_auto ships every input file there
          and submits to its queue — so you can turn this computer off. Results stay on the
          remote machine; fetch a copy back from the Results tab when the job finishes.
    
        
    No machines yet — add one on the right.
    
      
    
      
    
        
    Set up password-free SSH first
    
        
    vasp_auto connects non-interactively, so the remote must accept your
          SSH key without a password prompt. From a terminal:
    
        
    ssh-keygen -t ed25519        # once, if you have no key
ssh-copy-id user@host        # installs your key on the remote
ssh user@host 'echo ok'      # must print ok with no password
    
        
    Then use the same host/user below and press Test connection.
